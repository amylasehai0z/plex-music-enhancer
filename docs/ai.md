# AI Generation Subsystem

The AI layer generates album and artist descriptions from normalized context models. It is
provider-independent and is designed to be reused by Preview, Apply, Rollback, and future batch
processing.

## Architecture

The subsystem lives under:

```text
src/plex_music_enhancer/ai/
```

Key pieces:

- `AIProvider` defines the provider contract.
- `AIProviderRegistry` maps provider names to provider factories.
- `AIManager` loads the configured provider, validates it, exposes capabilities, and dispatches
  generation requests.
- `GeneratedSummary` is the normalized output model for all providers.
- `DummyProvider` provides deterministic local output.
- `OpenAIProvider` uses the official OpenAI Python SDK.

The AI manager consumes `AlbumContext` from the enrichment pipeline, builds a `RenderedPrompt`
through the prompt engine, and passes that prompt to the configured provider. It does not write to
Plex and does not call metadata providers directly.

## Provider Lifecycle

1. Configuration selects a provider name.
2. `AIManager` asks the registry to create that provider.
3. The provider validates its own configuration.
4. The manager exposes provider capabilities and metadata.
5. `AIManager` renders a prompt from the input context.
6. Generation requests are dispatched to the configured provider.

## Configuration

Default configuration:

```yaml
ai:
  provider: dummy
```

Environment override:

```text
PLEX_ENHANCER_AI__PROVIDER=dummy
```

## Providers

Supported provider names:

- `dummy`
- `openai`
- `ollama`

Implemented providers:

- `dummy`
- `openai`

`ollama` is a reserved provider name and will raise a clear availability error if selected.

## DummyProvider

`DummyProvider` is deterministic and performs no networking. It returns:

```text
This is a deterministic test summary generated by the DummyProvider for testing purposes.
```

It does not include timestamps or randomness inside generated text. The `created_at` timestamp is
stored in the structured `GeneratedSummary` metadata instead.

## Future Providers

Future providers should implement `AIProvider`, register themselves in the provider registry, and
return `GeneratedSummary` without leaking provider-specific response formats into the rest of the
application.
